Crear Crear un sistema de inicio de sesión: Si la contraseña es correcta, mostrar en el LCD “Correcto” y encender un LED. Si es incorrecto, encender una alarma y mostrar “Incorrecto” en el LCD.

1. Introducción

Este informe documenta el diseño, implementación y validación de un sistema de control de acceso basado en Arduino, que utiliza un teclado matricial 4x4 para el ingreso de una clave, una pantalla LCD 16×2 para la interfaz de usuario y un zumbador (buzzer) junto con dos LEDs como mecanismos de retroalimentación. El objetivo es permitir la autenticación mediante una contraseña de cuatro dígitos y activar señales acústicas y visuales según el resultado: verde en caso de acierto y rojo con alarma en caso de error.

2. Objetivos

  • Diseñar una interfaz de ingreso de clave con teclado 4×4 y LCD 16×2.
  • Implementar la verificación de contraseña con control de intentos fallidos.
  • Activar LED rojo y buzzer ante errores; LED verde en aciertos.
  • Incluir un modo de depuración que muestre dígitos ingresados durante pruebas.
  • Documentar el cableado, las decisiones de diseño y las pruebas realizadas.

3. Materiales y Herramientas

  • Arduino UNO (o equivalente)
  • Teclado matricial 4×4
  • Pantalla LCD 16×2 en modo paralelo
  • Buzzer activo o pasivo (función tone())
  • LEDs: Verde (acierto) y Rojo (error) + resistencias
  • Cables Dupont y protoboard
  • Librerías: LiquidCrystal, Keypad
  • IDE de Arduino 2.x

4. Metodología de Desarrollo

El desarrollo siguió un enfoque incremental:

  1. Definición del mapeo de pines y de la matriz de teclas del keypad.
  2. Configuración del LCD para visualización de mensajes y entrada.
  3. Captura de teclas con la librería Keypad y almacenamiento en buffer.
  4. Comparación con la clave correcta usando strcmp con terminador \0.
  5. Señalización con LED verde en acierto y LED rojo + buzzer en error.
  6. Conteo de intentos fallidos y alarma prolongada al tercer fallo.
  7. Modo de depuración para mostrar dígitos reales durante pruebas.

5. Lógica/Diagrama de Flujo (Descripción)

  • Si coincide: Mostrar “CLAVE CORRECTA”, encender LED verde, reiniciar contador de fallos.
  • Si no coincide: Mostrar “CLAVE INCORRECTA”, encender LED rojo y sonar buzzer; incrementar contador.
  • Si contador ≥ 3: Mostrar “ALARMA ACTIVADA”, mantener LED rojo, activar alarma prolongada y reiniciar contador.

    • 6. Ejecución del ejercicio

    7. Pruebas y Resultados

    • En modo depuración (SHOW_DIGITS=true), el sistema mostró correctamente los dígitos ingresados.
    • La LCD actualizó mensajes en tiempo real según el resultado (correcto/incorrecto/alarma).
    • LED verde se encendió únicamente en claves correctas; LED rojo y buzzer ante cada error.
    • Al tercer error consecutivo, la alarma prolongada funcionó y el contador se reinició.

    8. Discusión

    La librería Keypad simplifica la captura de teclas y evita gestionar rebotes manualmente. El LCD en modo paralelo permite feedback inmediato al usuario. Como mejora, se puede migrar a un módulo I2C para liberar pines, o evitar los pines 0 y 1 si se requiere comunicación serie. El modo de depuración acelera la validación de hardware y mapeo del teclado.

    9. Conclusión

    El sistema cumple los objetivos de control de acceso básico, manejo de intentos fallidos y señalización visual/auditiva. El modo de depuración facilita pruebas y garantiza que el diseño esté alineado con el código fuente implementado.
